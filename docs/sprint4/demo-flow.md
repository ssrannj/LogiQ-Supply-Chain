# Sprint 4 Demo Flow

## Objective
Demonstrate the end-to-end flow from route optimization to resilient data synchronization.

## Steps
1. **Dashboard Overview:** 
   - Show the current state of the supply chain dashboard.
   - Highlight the "Optimization Needed" status for pending shipments.

2. **Route Optimization:**
   - Trigger the route optimization action for a set of orders.
   - Show the "Before vs. After" comparison on the map/table.

3. **Real-time Notifications:**
   - Simulate a shipment status change.
   - Demonstrate the notification appearing in the UI (or logs).

4. **Replication Resilience:**
   - Simulate a node failure (if applicable/mocked).
   - Show that data remains consistent and accessible via replication.

5. **QA Summary:**
   - Quick overview of Sprint 4 test results and performance metrics.
